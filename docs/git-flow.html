GitLab CI/CD pipeline yapılandırması
💡 Branch prefixine göre pipeline davranışını ayarlamak, yanlış deploy riskini sıfıra indirir. Aşağıdaki örnekler .gitlab-ci.yml içine doğrudan eklenebilir.

✦ Feature & Bugfix Pipeline

Sadece build + test çalıştırır. Deploy tetiklenmez; yalnızca develop merge'ünde çalışır.

build:feature:
  stage: build
  only:
    - /^feature\/.*/
    - /^bugfix\/.*/
  script:
    - ./gradlew build test

deploy:
  stage: deploy
  only:
    - develop
  environment: staging
  script:
    - ./deploy.sh staging

⚡ Hotfix Pipeline

Hotfix branch'inde prod deploy tetiklenmez; yalnızca master merge'ünde manuel onay ile çalışır.

build:hotfix:
  stage: build
  only:
    - /^hotfix\/.*/
  script:
    - ./gradlew build test

deploy:production:
  stage: deploy
  only:
    - master
  when: manual
  environment: production
  script:
    - ./deploy.sh production

↩ Otomatik Back-Merge Job

Hotfix master'a merge olduktan sonra develop'a otomatik MR açar.

back-merge:to-develop:
  stage: .post
  only:
    - master
  script:
    - |
      curl --request POST \
        --header "PRIVATE-TOKEN: $GL_TOKEN" \
        --data "source_branch=master&
                target_branch=develop&
                title=back-merge" \
        "$CI_API_V4_URL/projects/\
         $CI_PROJECT_ID/merge_requests"

⬡ Staging Auto-Deploy

develop'a her merge'de staging otomatik güncellenir.

deploy:staging:
  stage: deploy
  only:
    - develop
  environment:
    name: staging
    url: https://staging.example.com
  script:
    - docker build -t app:$CI_COMMIT_SHA .
    - docker push registry/app:$CI_COMMIT_SHA
    - kubectl set image \
        deploy/app app=registry/app:\
        $CI_COMMIT_SHA
Conflict çözme — back-merge adımları
1
Hotfix branch'ini lokalde güncel hale getir
git checkout hotfix/issue-15610 git pull origin hotfix/issue-15610
2
develop'ı hotfix üzerine merge et (conflict'leri görmek için)
git fetch origin develop git merge origin/develop
3
Conflict işaretlerini bul ve çöz
# Conflict olan dosyaları listele git status # Editörde aç, <<<<<<< / ======= / >>>>>>> bloklarını çöz git add src/payment/PaymentService.java
4
Merge commit oluştur ve push'la
git commit -m "back-merge: master hotfix → develop (conflict resolved)" git push origin hotfix/issue-15610
5
GitLab'da develop hedefli MR aç ve merge et
# GitLab UI → Merge Requests → New Merge Request # Source: hotfix/issue-15610 # Target: develop # Title: [HOTFIX back-merge] Ödeme hatası → develop
⚠ Dikkat: Back-merge conflict'i erteleme. develop'ta bozuk kalırsa sonraki feature ve bugfix branch'leri aynı conflict'i miras alır. Hotfix master'a merge olduktan hemen sonra back-merge yapılmalı.
